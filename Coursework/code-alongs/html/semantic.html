Two Different Programming Styles

The two biggest programming styles today are Functional Programming and Object Oriented Programming. Both have their uses and practicioners of both tend to be very opinionated about which is better. In this article, we'll be taking a look and the differences between the two styles.

Functional Programming

Functional programming, as the name suggests, revolve around mathematical functions. A big part of the FP ethos is every function should return the same result given the same inputs. It also heavily discourages mutability/changing variables' values in order to uphold that ethos.

Object Oriented Programming

Object Oriented Programming is based around compartmentalizing concepts into real life "objects", where the inner data is hidden from the user and the objects instead provide a public interface to interact with them. Mutability is allowed. It works on the concepts of encapsulation and inheritance.
